1. Introduction

The Map Tile Viewer is a web-based tool designed to visualize map tiles from various sources. It allows you to:

  • Load and display map tiles from local or remote sources
  • Navigate through tile sets at different zoom levels
  • Inspect individual tile coordinates
  • Adjust viewing parameters in real-time

Key Features

  • Interactive pan and zoom navigation
  • Support for standard tile URL patterns
  • Adjustable tile size and zoom level
  • Coordinate display for tile inspection
  • Compact, hover-activated control panel

2. Getting Started

Basic Usage

  1. Hover over the left edge of the screen to reveal the control panel
  2. Enter your tile path/URL in the "Tile Path/URL" field
  3. Set the desired zoom level and tile coordinates
  4. Click "Generate Map" to load the tiles
  5. Use mouse drag to pan and mouse wheel to zoom

Tip: The tool comes pre-configured with OpenStreetMap's tile URL pattern as a default example.

3. Interface Overview

Control Panel

The collapsible control panel on the left contains all configuration options:

  • Tile Path/URL: The base path to your tile images
  • Tile Size: Pixel dimensions of each tile (typically 256 or 512)
  • Zoom Level: The map zoom level to display
  • X/Y Min/Max: The range of tile coordinates to load
  • Generate Map: Button to load tiles with current settings

Main View Area

The main display area shows the loaded tiles with these interactive elements:

  • Zoom Controls: +/- buttons in the top-right corner
  • Coordinates Display: Shows current tile coordinates when hovering
  • Status Message: Temporary loading/status notifications

4. Parameters Explained

Parameter Description Default/Example
Tile Path/URL Base URL or path to your tile images. Can include {z}, {x}, {y} placeholders. https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png
Tile Size Pixel dimensions of each square tile (width and height) 256
Zoom Level Map zoom level (higher numbers show more detail) 16
X Min/Max Range of tile column indices to load Varies by location
Y Min/Max Range of tile row indices to load Varies by location

Note: X and Y coordinates follow the Slippy Map tile naming convention.

5. Navigation Controls

Mouse Controls

  • Pan: Click and drag with left mouse button
  • Zoom In: Mouse wheel up or double-click
  • Zoom Out: Mouse wheel down or shift+double-click
  • Coordinates: Hover over tiles to see their X/Y indices

Button Controls

  • Zoom In (+): Top-right button or + key
  • Zoom Out (-): Bottom-right button or - key

Keyboard Shortcuts

  • + - Zoom in
  • - - Zoom out

6. Tile URL Patterns

The tool supports standard tile URL patterns with these placeholders:

Placeholder Description Example Replacement
{z} Zoom level 16
{x} Tile column index 12345
{y} Tile row index 9876
{s} Subdomain (for load balancing) a, b, or c

Example URL Patterns

  • OpenStreetMap: https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png
  • Local files: ./tiles/{z}/{x}/{y}.jpg
  • Custom server: https://maps.example.com/{zoom}/{col}/{row}.png

Important: When using local files, you may need to run a local web server due to browser security restrictions.

7. Troubleshooting

Common Issues

  • Tiles not loading:
    • Check your URL pattern is correct
    • Verify the server allows cross-origin requests (CORS)
    • Ensure the zoom level and coordinates exist
  • Blank tiles:
    • Check if the coordinates are valid for the zoom level
    • Verify the tile images exist at the specified path
  • Performance issues:
    • Reduce the number of tiles loaded at once
    • Use smaller tile sizes if possible

8. Frequently Asked Questions

How do I find the correct tile coordinates?

You can use online tools like MapTiler's coordinate calculator or convert from latitude/longitude using slippy map formulas.

Can I use this with my own tile server?

Yes, as long as your server provides tiles in the standard {z}/{x}/{y} format or a similar pattern you can adapt in the URL field.

Why can't I load local tile files directly?

Modern browsers restrict local file access for security. You'll need to either:

  • Run a local web server (like Python's python -m http.server)
  • Use browser extensions that relax these restrictions (not recommended for production)

What's the maximum number of tiles I can load?

There's no hard limit in the tool, but loading thousands of tiles may impact performance. For large areas, consider loading smaller ranges sequentially.
